# Getting started

This docs app ships in the workspace so you can preview the widget library against local source. Keep `pnpm` at the repo root and use the provided scripts so aliases point to `../../src` instead of the published package.

## Install the package

```bash
pnpm add @aomi-labs/widget-lib @assistant-ui/react @assistant-ui/react-markdown @tanstack/react-query @radix-ui/react-dialog @radix-ui/react-slot @radix-ui/react-avatar @radix-ui/react-separator @radix-ui/react-tooltip framer-motion motion lucide-react react-shiki remark-gfm tailwindcss zustand
```

## Import styles

Add the library stylesheet to your app entrypoint (or to `apps/docs/app/globals.css` when working locally):

```ts
import "@aomi-labs/widget-lib/styles.css";
```

Tokens live in `src/themes/default.css`. Override `--font-*`, `--radius-*`, or the palette in your host app to match your brand.

## Run the docs

The docs app lives at `apps/docs` and uses the library source directly.

```bash
pnpm run dev:docs
```

Then open `http://localhost:3005`. Turbopack will pick up changes in `src/**` automatically because `@source` in `globals.css` points at `../../src`.

## Mock data vs backend

Live previews render with mock data by default. To point at a backend, supply:

```env
NEXT_PUBLIC_PROJECT_ID=demo-project
NEXT_PUBLIC_BACKEND_URL=http://localhost:3001
```
