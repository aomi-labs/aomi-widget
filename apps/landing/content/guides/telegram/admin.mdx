# Bot Administration

Configure the Aomi Telegram bot using a TOML configuration file. This controls DM/group policies, wallet features, streaming, and transport mode.

## Configuration File

The bot reads its configuration from a TOML file (typically `bot.toml`):

```toml
enabled = true
bot_token = "YOUR_BOT_TOKEN"
dm_policy = "open"
group_policy = "mention"
mini_app_url = "https://your-app.com/telegram-wallet"
streaming = true
server_wc_enabled = false
# allow_from = [123456789, 987654321]
```

## Configuration Options

### enabled

Enable or disable the bot entirely.

```toml
enabled = true   # Bot is active
enabled = false  # Bot ignores all messages
```

### bot_token

Your Telegram Bot API token, obtained from [BotFather](https://t.me/BotFather).

```toml
bot_token = "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz"
```

### dm_policy

Controls who can message the bot in direct messages.

| Value | Behavior |
| --- | --- |
| `"open"` | Anyone can DM the bot |
| `"allowlist"` | Only users in `allow_from` can DM |
| `"disabled"` | Bot ignores all DMs |

```toml
dm_policy = "open"
```

### group_policy

Controls how the bot responds in group chats.

| Value | Behavior |
| --- | --- |
| `"mention"` | Responds only when mentioned (e.g., `@bot_name help`) |
| `"always"` | Responds to every message in the group |
| `"disabled"` | Bot ignores all group messages |

```toml
group_policy = "mention"
```

### allow_from

Telegram user IDs allowed to interact with the bot when `dm_policy` is set to `"allowlist"`. Specified as an array of integers.

```toml
allow_from = [123456789, 987654321]
```

### mini_app_url

The HTTPS URL for the Telegram mini-app used for wallet connection and transaction signing. Must start with `https://` (or `http://localhost` for development).

```toml
mini_app_url = "https://your-app.com/telegram-wallet"
```

### streaming

Enable or disable streaming responses. When enabled, the bot updates its message in real time as the AI generates text. When disabled, the bot sends a single message after the full response is ready.

```toml
streaming = true
```

### server_wc_enabled

Enable server-side WalletConnect flow for external wallets, as an alternative to the mini-app.

```toml
server_wc_enabled = false
```

## Transport Modes

The bot supports two transport modes for receiving updates from Telegram.

### Long Polling (Default)

When no `[webhook]` section is present, the bot uses long polling. This is simpler to set up and works without a public URL.

```toml
# No [webhook] section = long polling mode
enabled = true
bot_token = "YOUR_TOKEN"
```

### Webhook

For production deployments, webhooks are more efficient. Telegram sends updates directly to your server via HTTPS POST requests.

```toml
[webhook]
url = "https://bot.example.com/webhook"
addr = "0.0.0.0:8443"
```

| Field | Description |
| --- | --- |
| `url` | Public HTTPS URL that Telegram will POST updates to |
| `addr` | Local bind address for the webhook HTTP server |

The `url` must be publicly accessible and use HTTPS. Use a reverse proxy (nginx, Cloudflare Tunnel, ngrok) to terminate TLS if needed.

## Example Configurations

### Open Bot (Development)

```toml
enabled = true
bot_token = "YOUR_TOKEN"
dm_policy = "open"
group_policy = "mention"
mini_app_url = "http://localhost:3000"
streaming = true
```

### Restricted Bot (Production)

```toml
enabled = true
bot_token = "YOUR_TOKEN"
dm_policy = "allowlist"
group_policy = "mention"
mini_app_url = "https://app.yourcompany.com/wallet"
streaming = true
allow_from = [111111111, 222222222]

[webhook]
url = "https://bot.yourcompany.com/webhook"
addr = "0.0.0.0:8443"
```

### Group-Only Bot

```toml
enabled = true
bot_token = "YOUR_TOKEN"
dm_policy = "disabled"
group_policy = "always"
mini_app_url = "https://app.yourcompany.com/wallet"
streaming = true
```
