# Quickstart

Get a working AI chat widget running in your Next.js app. This guide assumes you already have an API key from Aomi.

## Prerequisites

- **Next.js 15** app with the App Router
- **Tailwind CSS** configured
- **shadcn/ui** initialized (`npx shadcn init` already run)

If you are starting from scratch:

```bash
npx create-next-app@latest my-app --typescript --tailwind --app
cd my-app
npx shadcn init
```

## Step 1: Install the Widget

```bash
npx shadcn add https://aomi.dev/r/aomi-frame.json
```

This installs:
- The `AomiFrame` component and all its sub-components
- The `@aomi-labs/react` runtime package
- shadcn UI primitives (button, sidebar, tooltip, etc.)
- `@assistant-ui/react` for the underlying chat primitives

## Step 2: Set Environment Variables

Create or update your `.env.local`:

```bash
# Required: URL of the Aomi backend
NEXT_PUBLIC_BACKEND_URL=https://api.aomi.dev
```

If you need wallet integration, also add:

```bash
# Optional: Reown project ID for wallet connect
NEXT_PUBLIC_PROJECT_ID=your_reown_project_id
```

## Step 3: Add AomiFrame to a Page

```tsx
import { AomiFrame } from "@/components/aomi-frame";

export default function ChatPage() {
  return (
    <div style={{ height: "100vh" }}>
      <AomiFrame
        backendUrl={process.env.NEXT_PUBLIC_BACKEND_URL!}
        height="100%"
        width="100%"
      />
    </div>
  );
}
```

## Step 4: Configure Your API Key

You have two options for setting the API key:

### Option A: Via the ControlBar UI

The widget includes a ControlBar with an API key input. Click the key icon, paste your API key, and select your namespace from the dropdown. The key is stored in `localStorage` for subsequent visits.

### Option B: Programmatically

Set the API key and namespace in code:

```tsx
import { AomiFrame } from "@/components/aomi-frame";

export default function ChatPage() {
  return (
    <AomiFrame
      backendUrl={process.env.NEXT_PUBLIC_BACKEND_URL!}
      height="100%"
    />
  );
}
```

Then use the `useControl` hook in a child component to set state programmatically:

```tsx
import { useControl } from "@aomi-labs/react";
import { useEffect } from "react";

function ConfigureOnMount() {
  const { setState } = useControl();

  useEffect(() => {
    setState({
      apiKey: "sk-your-api-key",
      namespace: "mycoindex",
    });
  }, []);

  return null;
}
```

## Step 5: Run Your App

```bash
npm run dev
```

Open `http://localhost:3000` (or wherever your page is mounted). You should see the full chat interface with:

- A sidebar for thread management
- A message composer at the bottom
- A ControlBar with model and namespace selectors
- Streaming AI responses when you send a message

## What You Should See

When you type a message and send it:

1. The message appears in the chat thread.
2. The assistant streams its response in real-time.
3. If the assistant calls a tool, you see the tool call and its result inline.
4. The response completes and the composer is ready for the next message.

## Customizing the Layout

Hide controls you do not need:

```tsx
<AomiFrame.Root height="100%" backendUrl={process.env.NEXT_PUBLIC_BACKEND_URL!}>
  <AomiFrame.Header
    withControl={true}
    controlBarProps={{
      hideWallet: true,    // No wallet connect
      hideApiKey: true,    // API key set programmatically
    }}
  />
  <AomiFrame.Composer />
</AomiFrame.Root>
```
