# Frontend Integration

Aomi offers two integration paths for adding an AI assistant to your frontend. Both connect to the same backend API and authentication system, so you can switch between them or combine them as needed.

## Integration Paths

### 1. Widget (shadcn Registry)

Pre-built UI components installed via the shadcn CLI. You get a complete chat interface out of the box, with the ability to edit every line of source code.

```bash
npx shadcn add https://aomi.dev/r/aomi-frame.json
```

**Best for:** Quick integration, standard chat UI, projects already using shadcn/ui. You want a working chat widget in minutes and can customize by editing the copied source files.

### 2. Headless Library

Pure logic distributed as an npm package. Hooks, providers, API client, and utilities with zero UI opinions.

```bash
npm install @aomi-labs/react
```

**Best for:** Fully custom designs, non-standard layouts, non-React frameworks (via the API client directly), or when you need maximum control over every pixel.

## Comparison

| | Widget (shadcn) | Headless Library |
| --- | --- | --- |
| **Setup time** | Minutes | Hours |
| **Install method** | `npx shadcn add` | `npm install` |
| **UI included** | Full chat interface, sidebar, controls | None |
| **Customization** | Edit copied source files | Build from scratch |
| **Dependencies** | `@aomi-labs/react`, `@assistant-ui/react`, shadcn primitives | `@aomi-labs/react`, `@assistant-ui/react` |
| **Framework** | Next.js (React) | Any React app (API client works anywhere) |
| **Code ownership** | Components copied to your project | Library imported from npm |

## Shared Foundation

Both paths use the same core:

- **`@aomi-labs/react`** provides all runtime logic: `AomiRuntimeProvider`, `useAomiRuntime`, `useControl`, `useUser`, and the `BackendApi` client.
- **Same backend API** endpoints for chat, threads, models, namespaces, and wallet operations.
- **Same authentication** via API keys and wallet addresses.

The widget is built on top of the headless library. When you install the widget, `@aomi-labs/react` is installed as a dependency automatically.

## Choosing a Path

**Start with the widget** if you want to see results quickly. Since the source code is copied into your project, you can always customize it later or replace individual components with your own.

**Start with the headless library** if you have an existing design system and want to integrate Aomi logic into your own components from day one.
