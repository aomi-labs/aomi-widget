# Use Aomi Quickstart

This quickstart helps end users get productive quickly.

## 1. Choose a Channel

- **Web**: open the app page where Aomi is embedded.
- **Telegram**: open your product's Aomi bot and run `/start`.

## 2. Set Your Working Context

- Select the namespace/environment if prompted.
- Select a model if your product exposes model choices.
- Connect wallet if you need transaction-enabled actions.

## 3. Run Your First Task

Try one of these:

- "Show my portfolio summary"
- "Get a quote to swap 1 ETH to USDC"
- "Summarize market moves in the last 24h"

## 4. Approve Requests Safely

When Aomi asks for wallet approval:

1. Verify amount, token, destination, and chain.
2. Confirm only if all details match your intent.
3. Reject and ask Aomi to regenerate if anything looks wrong.

## 5. Keep Sessions Organized

- Use separate threads for different goals.
- Rename important threads for easier retrieval.
- Continue work from the same channel when possible.
