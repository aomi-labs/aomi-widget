# Telegram Integration

Aomi provides a Telegram bot that gives users access to the same AI assistant available on the web. It works in both direct messages and group chats, with streaming responses, inline keyboard navigation, and wallet connectivity through a Telegram mini-app.

## How It Works

The Telegram bot connects to the same Aomi backend as web integrations. Messages sent to the bot are processed by the same runtime, tools, and LLM providers. This means:

- Users get the same AI capabilities in Telegram as on your website
- Conversation history is linked to the user's wallet address across platforms
- All configured namespaces, models, and tools are available

## Features

- **Streaming responses** -- AI responses stream in real time as they are generated, similar to the web chat experience
- **Inline keyboard panels** -- Interactive menus for managing settings, switching namespaces, selecting models, and viewing session history
- **Mini-app wallet** -- Connect a wallet through a Telegram WebApp (mini-app) for transaction signing and on-chain operations
- **DM and group chat support** -- Works in private conversations and group chats with configurable policies

## Direct Messages vs Group Chats

In **direct messages**, the bot responds to every message from the user (subject to DM policy configuration).

In **group chats**, the bot responds only when mentioned by name (e.g., `@your_bot help me swap ETH`) unless the group policy is set to respond to all messages. This prevents the bot from being noisy in active groups.

## Architecture

```
Telegram User
    │
    ▼
Telegram Bot (Rust / teloxide)
    │
    ├── Panel Router (slash commands + inline keyboards)
    │       ├── Start Panel
    │       ├── Namespace Panel
    │       ├── Model Panel
    │       ├── Wallet Panel
    │       ├── Sessions Panel
    │       ├── Network Panel
    │       ├── Settings Panel
    │       └── Transaction Panel
    │
    ├── Message Handler (DM / Group routing)
    │
    └── Aomi Runtime (same backend as web)
            ├── LLM Providers
            ├── Tool Scheduler
            └── Session Manager
```

The bot uses the panel router pattern: each feature (wallet, namespace, model, etc.) is a self-contained panel that handles its own slash commands, inline keyboard callbacks, and text input.
