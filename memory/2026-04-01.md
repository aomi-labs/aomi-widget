## Para auth sync on landing

- Fixed the landing Para wiring so the non-blocking docs app and the widget share the same `ParaWeb` instance. `apps/landing/src/components/wallet-providers.tsx` now initializes one client, uses it for `createParaWagmiConfig`, passes it into sibling `ParaProvider`, and exposes it on `window.__AOMI_PARA_CLIENT__`.
- Kept the landing app root wrapped in `apps/landing/app/provider.tsx` so the Para modal infrastructure is always mounted, while the hero/demo/playground frames continue to render normally.
- Hardened `apps/registry/src/lib/use-account-identity.ts` to read Para auth state outside the provider tree and to resolve the current address through `client.getAddress()` in addition to wallet maps.
- Verified live on `https://thiago-postdysenteric-impedingly.ngrok-free.dev`: the widget renders, `Connect Account` opens the Google/Para auth popup, and the previous `External wallet provider interface for type EVM is not initialized` warning no longer appears.
- Verification run: `pnpm --filter landing build`, `pnpm --filter @aomi-labs/widget-lib build`.

## Adapter boundary for registry widget

- Added `apps/registry/src/components/aomi-adapter-provider.tsx` as the new wallet adapter boundary and default Para-backed provider for registry consumers.
- Renamed `apps/registry/src/lib/use-account-identity.ts` to `apps/registry/src/lib/account-identity.ts` and made identity consumption adapter-based.
- Refactored registry `wallet-connect`, `network-select`, `wallet-tx-handler`, and `aomi-frame` to consume `AomiAdapterProvider` instead of importing Para/wagmi hooks directly.
- Updated `apps/registry/src/index.ts` and `apps/registry/src/registry.ts` to export and register the adapter provider, then regenerated `apps/registry/dist/registry.json` and `apps/registry/dist/aomi-frame.json`.
- Verification: `aomi-widget: pnpm build:registry`, `aomi-widget: pnpm lint`.
