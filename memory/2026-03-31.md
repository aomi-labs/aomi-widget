- replaced the landing/demo wallet provider stack from Reown AppKit to `@getpara/react-sdk`, matching the product-mono Para integration model
- added a shared registry-side `use-account-identity` hook and switched the widget `WalletConnect` control to Para modal flows (`AUTH_MAIN` / `ACCOUNT_MAIN`) while keeping wagmi for transaction execution/signing
- aligned the landing/demo chain configuration with the widget-supported chains, including Linea + Linea Sepolia and localhost support
- updated builder-facing docs and environment variable guidance from Reown/AppKit setup to Para provider setup
- removed direct Reown package dependencies from the active landing/root workspace configuration and rebuilt the registry payloads
- verified with `pnpm --filter @aomi-labs/widget-lib build`, `pnpm --filter landing build`, and `pnpm test`

## Landing Para hydration fix

- debugged a live blank-page regression in `apps/landing` after the Para migration using the dev server and Playwright
- identified the cause as the global `ParaProvider` boundary gating all children on Para readiness, which prevented the docs/landing shell from hydrating when the Para client never reached `isReady`
- removed the global Para wrapper from the root app provider and scoped `ContextProvider` to the actual `AomiFrame` surfaces in the home hero, widget demo, and playground configurator
- verified the homepage and `/docs/build/getting-started` render again in the live browser, with `pnpm --filter landing build` still passing
