## Client chain-id resilience (CLI + session)

- Added `UserState.reconcile()` in `packages/client/src/types.ts` to preserve wallet `address`/`chain_id` across partial snapshots and to drop invalid `is_connected: true` states when `chain_id` is missing.
- Updated `ClientSession` to use reconciliation for initial/session-applied user state and for backend alignment checks, preventing chain-id loss on partial backend `user_state` responses.
- Hardened CLI wallet-state flows:
  - `buildCliUserState()` now only marks `is_connected` when both `address` and `chain_id` are present.
  - `syncPendingTxsFromUserState()` no longer wipes persisted `chainId`/`publicKey` when backend omits those fields.
  - chat wallet state broadcast now requires a known `chainId`.
- Added/updated regression coverage in:
  - `packages/client/test/session.unit.test.ts`
  - `packages/client/test/cli/cli-chat.unit.test.ts`
  - `packages/client/test/cli/cli-session.unit.test.ts`
- Verification:
  - `pnpm vitest run packages/client/test/session.unit.test.ts`
  - `pnpm vitest run packages/client/test/cli/cli-chat.unit.test.ts packages/client/test/cli/cli-session.unit.test.ts`
  - `pnpm vitest run packages/client/test/cli/*.unit.test.ts packages/client/test/session.unit.test.ts`
